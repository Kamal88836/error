# 📌 What Is Happening Internally

Your `dpp` container has:

```yaml

  exec [cat /var/run/probe/swm_read_probe]
```

Kubernetes runs:

```
cat /var/run/probe/swm_read_probe
```

If:

* File exists → success → Pod becomes Ready
* File missing → failure → Pod stays NotReady

In telco/SR-IOV + hugepage workloads like yours:

### 1️⃣ Application inside container did not finish initialization

Most common.

### 2️⃣ HugePages allocation issue

You are requesting:

```
hugepages-1Gi: 46Gi
```

If not fully available → app may fail internally.

### 3️⃣ isolcpus (CPU pinning) not available

```
windriver.com/isolcpus: 44
```

If not satisfied correctly → app may hang.

### 4️⃣ SR-IOV VF not bound correctly

You are attaching **8 PCI devices**:

```
intel.com/pci_sriov_net_fh0
...
intel.com/pci_sriov_net_fh5
```

If VFIO binding failed → app startup fails silently.
